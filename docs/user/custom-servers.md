# Добавление своих серверов и прокси

Проект позволяет добавлять собственные VPN-серверы и Telegram-прокси через конфигурационные файлы. Они будут обработаны, отфильтрованы и объединены с остальными источниками автоматически.

## VPN-серверы

Отредактируйте файл `source/config/servers.txt`:

```
vless://uuid@host:52496?... # собственный VLESS-сервер
vmess://eyJhZGQiOiI...      # VMess в формате base64
trojan://password@host:443?...
```

**Формат:** одна строка = один конфиг. Поддерживаются все протоколы:

- `vless://`, `vmess://`, `trojan://` — основные
- `ss://`, `ssr://` — Shadowsocks
- `tuic://` — TUIC
- `hysteria://`, `hysteria2://`, `hy2://` — Hysteria

**Что произойдёт при запуске генератора:**

1. Ваши конфиги объединятся с конфигами из URL-источников
2. Пройдут фильтрацию безопасности (insecure-параметры будут отмечены)
3. Пройдут SNI/CIDR-фильтрацию для bypass-наборов
4. Попадут во все выходные файлы: номерные, all.txt, all-secure.txt, bypass, split-by-protocols

**Важно:** после 3 последовательных неудачных верификаций подряд конфиг автоматически удаляется из `servers.txt`.

## Telegram-прокси

Отредактируйте файл `source/config/tg_proxies.txt`:

**Поддерживаемые форматы:**

| Формат | Пример |
|--------|--------|
| MTProto | `https://t.me/proxy?server=1.2.3.4&port=443&secret=...` |
| SOCKS5 | `https://t.me/socks?server=1.2.3.4&port=1080&user=...&pass=...` |
| tg:// MTProto | `tg://proxy?server=1.2.3.4&port=443&secret=...` |
| tg:// SOCKS5 | `tg://socks?server=1.2.3.4&port=1080` |
| Прямой SOCKS5 | `socks5://1.2.3.4:1080` |

**Что произойдёт:**

1. Ваши прокси объединятся со скрапированными из источников
2. Пройдут верификацию (проверка работоспособности)
3. Будут отсортированы по пингу (самые быстрые — первыми)
4. Попадут в `tg-proxy/all.txt`, `MTProto.txt` и `socks.txt`

## Скрапинг Telegram-прокси из URL

Прокси извлекаются из контента URL-источников через `TelegramProxyScraper`. Используются регулярные выражения для 4 форматов MTProto и 6 форматов SOCKS5:

| Тип | Паттерн |
|-----|---------|
| MTProto | `https://t.me/proxy?...`, `http://t.me/proxy?...`, `t.me/proxy?...` (bare), `tg://proxy?...` |
| SOCKS5 | `https://t.me/socks?...`, `http://t.me/socks?...`, `t.me/socks?...`, `tg://socks?...`, `socks5://host:port`, `IP:PORT` bare format |

Извлечённые прокси проходят дедупликацию и очистку от эмодзи/мусора.

## Принцип работы

Все файлы обрабатываются тем же пайплайном, что и внешние источники — к ним применяются все фильтры, дедупликация и верификация.

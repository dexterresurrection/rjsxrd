# Прокси-цепочки

Поддержка многоуровневого маршрутизирования через Xray `dialerProxy`. Позволяет направить трафик через цепочку из нескольких VLESS-серверов в рамках одного процесса Xray.

**Статус:** экспериментальная функция.

## Архитектура

```
Приложение → Xray (SOCKS5 :22000)
                  ↓
             VLESS Hop 1 (chain-0)
                  ↓ dialerProxy="chain-1"
             VLESS Hop 2 (chain-1)
                  ↓
              Интернет
```

## Использование

```bash
python main.py --proxy-chain="vless://hop1,vless://hop2,vless://hop3"
```

**Требования:**
- Минимум 2 прокси в цепочке
- Все прокси должны быть одного типа (VLESS или VMess)

### Поддерживаемые транспорты

| Транспорт | Совместимость |
|-----------|---------------|
| VLESS + WebSocket + TLS | ✅ Полная |
| VLESS + HTTPUpgrade + TLS | ✅ Полная |
| VMess + WebSocket + TLS | ✅ Полная |
| VLESS + Reality | ❌ Несовместим с dialerProxy |
| VLESS + TCP | ⚠️ Возможны проблемы |

**Важно:** Reality не работает с `dialerProxy` из-за особенностей транспортного механизма на уровне сокетов.

## Формат URL для совместимых конфигов

```
vless://uuid@example.com:443?encryption=none&security=tls&type=ws&host=example.com&path=%2F
```

## Реализация

Функция `create_chain_config()` в `source/utils/xray_tester.py`:

1. Реверсирует порядок hop'ов для корректной маршрутизации
2. Генерирует Xray-конфиг с несколькими outbound'ами и `dialerProxy`
3. Запускает один Xray-процесс с единым SOCKS5-входом
4. Мониторит стабильность соединения в фоновом потоке (каждые 30 с)

## Ограничения

- Первый hop видит ваш реальный IP
- Reality-протокол не поддерживается
- Функция экспериментальная, возможны проблемы с отдельными серверами

## ProxyMonitor — мониторинг цепочки

При запуске прокси-цепочки создаётся `ProxyMonitor` в фоновом потоке:

- **Интервал проверок:** каждые 30 с
- **Что проверяет:** SOCKS-порт цепочки через HTTP-запрос к `https://www.gstatic.com/generate_204`
- **Сравнение IP:** запрашивает внешний IP через цепочку и сравнивает с реальным IP (без прокси)
- **Регистрация:** добавляется в глобальный `_active_proxy_monitors` для корректного graceful shutdown
- **Остановка:** при завершении `main.py` монитор останавливается до уничтожения Xray-процессов

## Ограничения

**Цепочка выходит только через первый hop:**
Проверьте транспорт — вероятно, используется Reality. Переключитесь на WebSocket.

**Соединение обрывается сразу:**
Проверьте путь (`path`) и хост (`host`) в каждом URL. Убедитесь, что TLS включён (параметр `security=tls`).

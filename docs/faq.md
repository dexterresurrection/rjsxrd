# Часто задаваемые вопросы

## О проекте

**Что такое rjsxrd?**  
Автоматически обновляемая коллекция публичных VPN-конфигов. Проект собирает, фильтрует и верифицирует конфиги из открытых источников, обновляя их **каждый час с VPS** (основной, cron) и **раз в 2 дня** через GitHub Actions (резервный).

**Откуда берутся конфиги?**  
Из публичных источников, перечисленных в `source/config/URLS.txt`. Проект не создаёт свои серверы — только агрегирует и проверяет существующие.

**Насколько это безопасно?**  
Конфиги проходят фильтрацию небезопасных параметров и верификацию. Но это публичные конфиги — используйте с пониманием рисков. Для конфиденциальных данных рекомендуется собственный VPN.

## Использование

**Почему конфиги разделены по 300 строк в файле?**  
Ограничение GitHub на размер файла (~49 MB). Разделение также ускоряет импорт на мобильных устройствах.

**Что выбрать: default или bypass?**  
- `default/` — для обычных блокировок
- `bypass/` — если мобильный оператор блокирует VPN по SNI/IP

**В чём разница bypass и bypass-unsecure?**  
- `bypass/` — только безопасные конфиги (proшли insecure-фильтрацию)
- `bypass-unsecure/` — все конфиги, включая небезопасные

**Зачем нужна папка split-by-protocols?**  
Если вы знаете, какой протокол вам нужен (например, только VLESS), можно взять файл оттуда, а не качать все конфиги.

**Как добавить свой сервер?**  
Поместите его в `source/config/servers.txt`. При следующем запуске генератора он будет обработан и объединён с остальными.

**Как добавить Telegram-прокси?**  
Поместите их в `source/config/tg_proxies.txt`. Поддерживаются форматы `https://t.me/proxy?...`, `https://t.me/socks?...`, `tg://proxy?...`, `tg://socks?...`.

## Режимы генератора

**В чём разница между --skip-xray, --tcp-ping и обычным запуском?**  
- **Обычный:** Xray-core запускает процесс на каждый конфиг, делает HTTP-запрос. Максимально точный, но самый медленный (60-120 с).
- **--tcp-ping:** TCP-соединение к host:port. Быстрый (~5 с), проверяет только открыт ли порт. Не требует Xray.
- **--skip-xray:** Вообще без проверки. Самый быстрый, но конфиги могут быть нерабочими.

**Как запустить генератор без загрузки в GitHub?**  
```bash
python main.py --dry-run
```

**Что такое --use-git?**  
Режим для GitHub Actions. Использует git-команды вместо GitHub API для загрузки файлов.

## Проблемы

**Не работает ни один конфиг. Что делать?**  
1. Перезапустите приложение и устройство
2. Проверьте брандмауэр и антивирус
3. Обновите список конфигов в приложении
4. Попробуйте другой файл (например, bypass-all.txt вместо 1.txt)

**Медленное подключение**  
Выберите конфиг с меньшим пингом. Попробуйте другой протокол. Проверьте качество интернет-соединения.

**Проблемы с импортом на мобильном**  
- Сначала попробуйте `bypass-all.txt`
- Если тормозит — используйте разделённые файлы (bypass-1.txt и т.д.)
- Обновите VPN-клиент до последней версии

**При импорте конфиги не отображаются**  
В v2rayNG: «три полоски» → «Группы» → «иконка кружка со стрелкой» (обновить подписку).
